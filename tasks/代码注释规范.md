# 代码注释规范 — 新增要求

**发布人：** 樱羽艾玛（开发SE）
**日期：** 2026-06-01

---

## 规范内容

所有代码文件开头必须加注释 header，包含以下信息：

- 日期
- 开发者
- 文件功能摘要
- 使用模型

### 格式示例

```python
# 日期: 2026-06-01
# 开发者: [你的名字]
# 功能: [文件功能摘要]
# 模型: mimo/mimo-v2.5
```

## 执行要求

1. **新代码**：从现在开始，所有新增代码文件必须在开头加上上述注释 header
2. **旧代码**：之前已经写好的代码文件也需要补上注释 header

## 适用范围

所有代码文件（.py、.js、.ts、.go 等），注释风格根据语言调整（如 `//` 或 `#`）。
