### 2. 具体实现要求

#### main.py
- FastAPI应用，暴露 POST `/onebot` 端点
- 启动时初始化SQLite缓存（调用cache模块）
- 启动时拉取menu树并构建operation_index

#### config.py
- 从 `.env` 读取配置项：
  - `NAPCAT_URL`：NapCat地址，默认 `http://napcat:3001`
  - `FASTAPI_PORT`：FastAPI端口，默认 `8080`
  - `WIKI_BASE_URL`：wiki API地址，默认 `https://wiki.arkrec.com`
  - `WIKI_API_TOKEN`：API认证token
  - `DEFAULT_CATEGORY`：默认流派，一期暂不使用（所有指令流派必填）
  - `STALE_DAYS`：待压人天数阈值，默认 `365`

#### qq_bot/onebot.py
- 解析onebot11事件格式，提取：
  - `post_type`：消息类型
  - `message_type`：群消息/私聊
  - `group_id`：群号
  - `user_id`：用户ID
  - `message`：消息内容

#### qq_bot/handler.py
- 指令前缀：`#`
- 正则匹配规则：
  ```python
  COMMANDS = {
      "query":     r"#查\s+(.+)",
      "uncleared": r"#未通关\s+(.+)",
      "stale":     r"#待压人\s+(.+)",
      "category":  r"#流派\s*",
      "help":      r"#help",
  }
  ```
- 将匹配到的参数传递给对应command
- 未匹配到任何指令时不回复

#### qq_bot/commands/*.py
- 一期先实现空壳，接收参数后返回占位消息
- 例如 query.py 返回：`[TODO] 查询关卡: {args}`
- help.py 返回固定帮助文本：
  ```
  📋 少人WIKI 低配低练记录查询Bot

  #查 <关卡> <流派> — 按关卡查最佳记录
  #未通关 <活动名> <流派> — 查未通关关卡
  #待压人 <活动名> <流派> [天数] — 查待压人记录（默认365天）
  #流派 — 查看所有流派
  #help — 显示本帮助

  示例：#查 H11-1 四星队
  ```

#### docker-compose.yml
```yaml
services:
  napcat:
    image: napcat/napcat:latest
    ports:
      - "3001:3001"
    volumes:
      - ./napcat-data:/app/napcat/config
    restart: unless-stopped

  fastapi:
    build: .
    ports:
      - "8080:8080"
    environment:
      - NAPCAT_URL=http://napcat:3001
      - WIKI_BASE_URL=https://wiki.arkrec.com
      - WIKI_API_TOKEN=${WIKI_API_TOKEN}
    depends_on:
      - napcat
    restart: unless-stopped
```

#### Dockerfile
- Python 3.11 slim基础镜像
- 安装依赖
- 暴露8080端口
- 启动命令：`uvicorn main:app --host 0.0.0.0 --port 8080`
